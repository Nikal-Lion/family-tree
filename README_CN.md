## 1. 产品概述

### 1.1 产品背景
传统纸质族谱查阅不便、更新困难，且难以在家族成员间共享。随着数字化普及，需要一个轻量级、跨平台的电子族谱系统，让家族成员能够便捷地记录、查看和管理家族世系，传承家族文化。

### 1.2 产品定位
一款面向家庭/家族使用的**轻量级电子族谱管理工具**，提供族谱树形可视化展示与后台管理功能，支持移动端和PC端自适应访问，支持 Cloudflare D1 云端存储与本地离线兜底，开箱即用。

### 1.3 产品目标
- 提供直观的族谱树状图，支持缩放拖拽，方便查看多代关系。
- 提供完整的成员管理功能（增、删、改），维护家族成员信息。
- 提供家族事件记录功能，记录婚丧嫁娶等重要时刻。
- 适配移动端与PC端，确保在不同设备上均有良好体验。
- 数据持久化至 Cloudflare D1 云端，支持多设备访问与离线备份。

## 3. 核心功能需求

### 3.1 族谱可视化展示（用户端/管理端共享）
- **树状图呈现**：使用 ECharts 库生成正交树形图，以"族谱总览"为虚拟根节点，所有无父亲记录的成员作为顶级分支。
- **交互操作**：
  - 支持鼠标/手指拖拽画布、双指缩放/滚轮缩放。
  - 支持节点折叠/展开（点击节点左侧的折叠图标）。
  - 鼠标悬停显示成员简要信息（姓名、性别、ID）。
  - 点击节点在右侧详情面板展示完整信息，并提供快捷编辑/删除入口。
- **搜索高亮**：输入关键词后，匹配的节点在树图中高亮显示。
- **自适应布局**：画布高度在 PC 端为 560px，移动端自动调整为 460px，保证可视区域。

### 3.2 成员管理（管理端）
#### 3.2.1 新增成员
- **字段**：姓名（必填）、父亲（可选，从现有成员中选择）、配偶（可选，可多选，可清空）、性别（男/女）、出生日期（可选）、照片 URL（可选）、生平简介（可选）。
- **逻辑**：
  - 若未选择父亲，则该成员作为新的始祖/分支，在树图中独立显示。
  - 添加成功后，自动刷新树图、成员列表及父级下拉框。
  - 防止添加空姓名，提交时校验。

#### 3.2.2 编辑成员
- **入口**：
  - 成员列表右侧的"编辑"按钮。
  - 详情面板中的"编辑"按钮。
- **可编辑字段**：姓名、父亲（支持重新指定父节点）、配偶（支持多选并可清空）、性别、出生日期、照片 URL、生平简介。
- **约束**：
  - 不能将父亲设为自己或自己的后代（循环依赖检测）。
  - 若新父亲无效（不存在），则设为无父亲。
  - 更新后刷新全UI。

#### 3.2.3 删除成员
- **入口**：成员列表右侧的"删除"按钮、详情面板中的"删除"按钮。
- **逻辑**：
  - 删除时将同时删除该成员及其所有后代（递归删除）。
  - 同时级联删除该成员关联的所有事件。
  - 弹出二次确认提示，防止误操作。
  - 删除后自动刷新数据。

#### 3.2.4 成员列表
- 以列表形式展示所有成员，按 ID 排序。
- 每一项显示：姓名、性别、父亲（若存在则显示"父:xxx"，否则显示"始祖"）。
- 点击列表项可查看详情。
- 列表支持滚动，移动端可滑动查看。

### 3.3 成员详情查看
- **触发方式**：点击树图中节点或成员列表项。
- **展示内容**：
  - 姓名、性别、父亲姓名（若存在）、成员 ID。
  - 配偶列表（支持多配偶显示）。
  - 出生日期、照片（支持 URL 展示与预览）、生平简介。
  - 内置"编辑"和"删除"按钮，可快捷操作。
- **默认状态**：页面加载时，自动展示第一个成员的详情。

### 3.3.1 成员搜索与高亮
- 在成员列表上方提供搜索框，支持按**姓名**或**ID**关键词搜索。
- 搜索结果在族谱树图中**高亮显示**匹配节点，便于快速定位。
- 清空搜索框后恢复正常显示。

### 3.4 数据持久化
- **主存储路径**：使用 Cloudflare D1 云端数据库存储所有成员、轨迹、事件数据，支持多设备访问。
- **兼容路径**：保留 localStorage / SQLite 作为离线兜底与历史兼容。
- 每次增删改操作后自动保存至云端。
- 页面启动时自动连接 D1 并恢复数据；连接失败时提示重试。
- 支持 JSON 和 SQLite 文件的导入导出，便于离线备份。

### 3.5 扫墓轨迹管理（Stage2 P0）
#### 3.5.1 轨迹上传与管理
- 支持上传 `.gpx` 文件并进行前端解析。
- 轨迹可设置名称，并可关联到具体成员（墓主）。
- 支持轨迹重命名与删除。

#### 3.5.2 轨迹展示与导航
- 管理区展示轨迹列表，包含名称、关联成员、距离、轨迹点数。
- 支持一键导航，跳转网页地图到轨迹终点坐标。
- 轨迹数据持久化保存，刷新页面后可继续使用。

### 3.6 族谱照片 OCR 导入（Stage2 P0）
#### 3.6.1 图片上传与预处理
- 支持上传族谱照片（jpg/png/heic），移动端支持调用相机拍照。
- 前端自动压缩图片（宽度上限 2000px）后再进行识别。

#### 3.6.2 OCR 识别与解析
- 调用 OCR 服务识别文字内容，提取姓名、父亲、母亲、配偶等字段。
- 采用规则解析将文本映射为可编辑的临时成员数据。

#### 3.6.3 预览编辑与去重导入
- 提供表格式预览工作台，支持增删改识别结果。
- 导入前提示重复候选，用户选择策略后批量导入族谱。

### 3.7 本地存储升级（SQLite 第一期，规划中）
#### 3.7.1 存储目标
- **当前状态**：主存储路径为 Cloudflare D1，连接失败时回退 localStorage。
- **阶段目标**：将本地主路径升级为 SQLite（SQL.js）+ IndexedDB 持久化。
- **阶段目标**：在保留 JSON 导入导出的同时，补齐真实 SQLite 二进制导入导出能力。

#### 3.7.2 兼容与迁移
- 已兼容历史 localStorage 数据：首次连接 D1 时可将本地数据作为种子数据写入云端。
- 支持降级回退：云端异常时回退 localStorage 保证可用性。
- 规划补齐：SQLite 启用后的迁移流程、双写策略与灰度开关。

#### 3.7.3 SQLite 核心模型（一期）
- members：成员主表（id/name/parent_id/gender/spouse_ids/created_at/updated_at）。
- tracks：轨迹表（points、起终点、统计信息 JSON 化存储）。
- events：事件表（id/member_id/type/title/date/description/created_at/updated_at）。
- metadata：元数据表（schema_version/next_id/next_track_id/next_event_id/last_sync_time 等）。

### 3.8 家族事件管理
#### 3.8.1 事件类型
- 支持七种预置事件类型：**婚**、**丧**、**嫁**、**娶**、**生**、**卒**、**其他**。
- 每个事件可关联一位成员，也可不关联（记录家族层面的公共事件）。

#### 3.8.2 事件 CRUD
- **新增事件**：填写事件类型、标题（必填）、日期（必填）、关联成员（可选）、说明（可选）。
- **编辑事件**：点击事件列表中的"编辑"按钮进入编辑态，修改后保存。
- **删除事件**：点击"删除"按钮并确认后移除事件记录。
- **级联清理**：删除成员时，同时删除该成员关联的所有事件。

#### 3.8.3 事件列表展示
- 事件按日期倒序排列，展示日期、类型、标题、关联成员名称。
- 支持事件说明的展示。

### 3.9 Toast 通知系统
- 所有操作反馈通过统一的 Toast 通知组件展示，替代部分原生 alert。
- 支持三种类型：**成功**（绿色）、**错误**（红色）、**信息**（蓝色）。
- 通知自动在 2.2 秒后消失，带淡入淡出动画效果。

## 7. 交互流程与约束

### 7.1 新增成员流程
1. 用户在右侧管理面板输入姓名、选择父亲（可选）、性别、出生日期、照片 URL、生平简介。
2. 点击"保存"按钮提交。
3. 系统校验姓名非空，校验通过后创建新成员并分配自增 ID。
4. 保存至 Cloudflare D1 云端，刷新树图、成员列表和父级选项。
5. 新增成功后自动选中新成员，便于继续查看或编辑。

### 7.2 编辑成员流程
1. 用户通过列表或详情进入编辑态，复用右侧管理面板（非弹窗）。
2. 修改姓名、父亲、性别或档案字段后点击"保存"。
3. 系统进行姓名非空、自引用和后代检测校验。
4. 校验通过则更新成员对象并刷新界面。
5. 若校验失败则提示错误原因并保持当前数据不变。

### 7.3 删除成员流程
1. 用户点击删除按钮。
2. 弹出确认框："确认删除 [姓名] 及其全部后代吗？预计删除 X 位成员。"。
3. 用户确认后，递归获取所有后代 ID，从成员数组中过滤删除（级联删除）。
4. 同时级联删除被删成员关联的所有事件。
5. 保存并刷新界面，提示"已删除 X 位成员"。

### 7.4 循环依赖检测规则
当编辑成员的父亲时，系统会检查：若将 A 的父亲设为 B，则 B 不能是 A 自身，也不能是 A 的后代。检测算法为基于子树遍历（DFS/栈）判断候选父节点是否位于 A 的后代集合中，若命中则拒绝操作。

### 7.5 JSON 导入导出流程
1. 用户点击"导入 JSON/SQLite"后选择本地 JSON 或 SQLite 文件。
2. 系统先弹出覆盖确认："导入将覆盖当前数据，是否继续？"。
3. 用户确认后执行导入；成功提示"导入成功"，失败提示具体错误。
4. 用户可点击"导出 JSON"或"导出 SQLite"下载当前数据备份。
5. 导入结束后会清空文件输入状态，避免重复触发。

### 7.5.1 SQLite 导入导出流程
1. 当前版本点击"导出 SQLite"会导出兼容备份文件（JSON 序列化，扩展名 `.sqlite/.db`）。
2. 用户可选择该备份文件导入，导入前执行覆盖确认。
3. 导入后系统替换当前数据并刷新树图、列表、详情。
4. 真实 SQLite 二进制导入导出能力列入 M2.5 计划。

### 7.5.2 首次迁移流程（localStorage -> SQLite）
1. 页面启动时优先初始化 SQLite 与 IndexedDB。
2. 若 SQLite 为空且检测到 localStorage 数据，则执行一次性迁移。
3. 迁移成功后写入迁移标记，并进入 SQLite 主路径。
4. 迁移失败时回退至 localStorage 并提示用户导出备份。

### 7.6 OCR 导入流程
1. 用户点击"从族谱照片导入"，上传图片或使用拍照入口。
2. 系统完成图片压缩并执行 OCR 识别。
3. 前端解析识别文本并展示可编辑预览表格。
4. 用户可修正字段并处理重复候选（跳过/新增）。
5. 点击导入后批量写入成员与关系并刷新树图。

### 7.7 事件管理流程
1. 用户在事件管理面板选择事件类型、填写标题和日期（必填），可选关联成员和说明。
2. 点击"新增事件"提交，系统校验标题和日期非空后创建事件。
3. 编辑事件：点击事件列表"编辑"按钮，表单切换为编辑态，修改后保存。
4. 删除事件：点击"删除"按钮，确认后移除事件记录。
5. 事件按日期倒序排列展示。

## 8. 技术实现说明

| 技术栈 | 用途 |
|-------|------|
| Vue 3 | 组件化界面与响应式状态驱动 |
| TypeScript | 类型约束与业务逻辑可靠性 |
| Vite | 本地开发与构建工具链 |
| ECharts ^5.6.0 | 族谱树可视化渲染与交互 |
| Cloudflare D1 | 云端关系型数据库，主存储路径 |
| Cloudflare Workers | D1 数据库的 API 代理层 |
| Cloudflare Pages | 前端静态资源托管与部署 |
| SQL.js + SQLite | 结构化本地数据存储（离线兜底） |
| IndexedDB | SQLite 数据库二进制持久化 |
| localStorage | 兼容迁移与回滚兜底 |

### 关键技术点
- **树图数据转换**：将扁平成员数组转换为嵌套的树结构，通过虚拟根节点统一挂载。
- **搜索高亮**：搜索关键词匹配成员后，将匹配 ID 集合传递至树图组件进行节点高亮渲染。
- **自适应**：使用 `@media` 查询配合 Flex 布局。
- **移动端触摸优化**：ECharts 默认支持触摸事件，通过 `roam: true` 启用缩放拖拽。
- **关系校验**：更新父节点时执行自引用与后代检测，防止关系成环。
- **配偶双向一致性**：添加/删除配偶时自动维护双向引用，确保数据一致。
- **事件级联删除**：删除成员时自动清理关联事件，防止悬挂引用。
- **导入导出闭环**：支持 JSON/SQLite 文件导入、覆盖确认与导出备份。
- **云端存储**：通过 Cloudflare Workers 代理 D1 API，前端异步持久化数据。
- **存储升级兼容**：支持 localStorage 到 SQLite 一次性迁移、双写过渡和降级回退。
- **Toast 通知**：统一的操作反馈系统，成功/错误/信息三种类型，自动消失。

## 9. 异常场景处理

| 导入 JSON 格式无效 | 提示"导入失败：JSON 格式无效" |
| 导入数据结构错误 | 提示"导入失败：数据结构错误" |
| 导入后无有效成员数据 | 提示"导入失败：未找到有效成员数据" |
| 导入数据成员 ID 重复 | 提示"导入失败：成员 ID 重复" |
| 导入覆盖确认未通过 | 取消导入，保持原数据不变 |

| SQLite 初始化失败 | 提示"本地数据库初始化失败"，支持重试 |
| SQLite 导入失败 | 提示"SQLite 导入失败"，保持原数据不变 |
| 首次迁移失败 | 回退 localStorage 并提示导出备份 |
| D1 云端连接失败 | 提示"云端 D1 数据初始化失败"，提供重试按钮 |
| 事件标题为空 | Toast 提示"事件标题不能为空"，不执行保存 |
| 事件日期为空 | Toast 提示"事件日期不能为空"，不执行保存 |
| 删除事件不存在 | 返回"事件不存在"，不执行删除 |
| 导航弹窗被拦截 | Toast 提示"导航页面被拦截，请允许弹窗后重试" |

## 附录：术语表

- **族谱**：记载家族世系和重要人物事迹的图册。
- **始祖**：家族中无父亲记录的成员，代表一个分支的起点。
- **虚拟根节点**：在树图中为了统一展示所有顶级成员而创建的非实际节点，不存储于数据中。
- **循环依赖**：将后代设为祖先的父亲，导致关系环路，属于逻辑错误。
- **家族事件**：家族中的重要时刻记录，如婚丧嫁娶、出生、逝世等。
- **世代**：成员在家族树中的层级深度，始祖为第 1 代，其子女为第 2 代，依此类推。
