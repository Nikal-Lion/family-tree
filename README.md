## 1. 产品概述

### 1.1 产品背景
传统纸质族谱查阅不便、更新困难，且难以在家族成员间共享。随着数字化普及，需要一个轻量级、跨平台的电子族谱系统，让家族成员能够便捷地记录、查看和管理家族世系，传承家族文化。

### 1.2 产品定位
一款面向家庭/家族使用的**轻量级电子族谱管理工具**，提供族谱树形可视化展示与后台管理功能，支持移动端和PC端自适应访问，数据本地存储，无需后端服务器即可独立运行。

### 1.3 产品目标
- 提供直观的族谱树状图，支持缩放拖拽，方便查看多代关系。
- 提供完整的成员管理功能（增、删、改），维护家族成员信息。
- 适配移动端与PC端，确保在不同设备上均有良好体验。
- 数据持久化于浏览器本地，无需注册登录，开箱即用。

## 2. 目标用户

| 用户角色 | 特征描述 | 核心需求 |
|---------|---------|---------|
| 家族管理员 | 负责族谱的维护与更新，通常为家族中较有精力、熟悉电脑/手机的成员 | 添加/编辑/删除成员信息，确保族谱准确；查看全貌 |
| 普通家族成员 | 浏览族谱，了解家族世系和成员关系 | 查看族谱树、成员详情，快速定位自己或亲属的位置 |
| 研究者/兴趣者 | 对家族历史感兴趣的外部人员（受限） | 浏览族谱结构，但无修改权限（当前版本无权限区分） |

## 3. 核心功能需求

### 3.1 族谱可视化展示（用户端/管理端共享）
- **树状图呈现**：使用 ECharts 库生成正交树形图，以“族谱总览”为虚拟根节点，所有无父亲记录的成员作为顶级分支。
- **交互操作**：
  - 支持鼠标/手指拖拽画布、双指缩放/滚轮缩放。
  - 支持节点折叠/展开（点击节点左侧的折叠图标）。
  - 鼠标悬停显示成员简要信息（姓名、性别、ID）。
  - 点击节点在右侧详情面板展示完整信息，并提供快捷编辑/删除入口。
- **自适应布局**：画布高度在 PC 端为 560px，移动端自动调整为 460px，保证可视区域。

### 3.2 成员管理（管理端）
#### 3.2.1 新增成员
- **字段**：姓名（必填）、父亲（可选，从现有成员中选择）、配偶（可选，可多选，可清空）、性别（男/女）。
- **逻辑**：
  - 若未选择父亲，则该成员作为新的始祖/分支，在树图中独立显示。
  - 添加成功后，自动刷新树图、成员列表及父级下拉框。
  - 防止添加空姓名，提交时校验。

#### 3.2.2 编辑成员
- **入口**：
  - 成员列表右侧的“编辑”按钮。
  - 详情面板中的“编辑”按钮。
- **可编辑字段**：姓名、父亲（支持重新指定父节点）、配偶（支持多选并可清空）、性别。
- **约束**：
  - 不能将父亲设为自己或自己的后代（循环依赖检测）。
  - 若新父亲无效（不存在），则设为无父亲。
  - 更新后刷新全UI。

#### 3.2.3 删除成员
- **入口**：成员列表右侧的“删除”按钮、详情面板中的“删除”按钮。
- **逻辑**：
  - 删除时将同时删除该成员及其所有后代（递归删除）。
  - 弹出二次确认提示，防止误操作。
  - 删除后自动刷新数据。

#### 3.2.4 成员列表
- 以列表形式展示所有成员，按 ID 排序。
- 每一项显示：姓名、性别、父亲（若存在则显示“父:xxx”，否则显示“始祖”）。
- 点击列表项可查看详情。
- 列表支持滚动，移动端可滑动查看。

### 3.3 成员详情查看
- **触发方式**：点击树图中节点或成员列表项。
- **展示内容**：
  - 姓名、性别、父亲姓名（若存在）、成员 ID。
  - 内置“编辑”和“删除”按钮，可快捷操作。
- **默认状态**：页面加载时，自动展示第一个成员的详情。

### 3.4 数据持久化
- 使用浏览器 localStorage 存储所有成员数据，键名为 `family_tree_app_data`。
- 每次增删改操作后自动保存。
- 页面刷新或重新打开时，自动加载上次保存的数据。

### 3.5 扫墓轨迹管理（Stage2 P0）
#### 3.5.1 轨迹上传与管理
- 支持上传 `.gpx` 文件并进行前端解析。
- 轨迹可设置名称，并可关联到具体成员（墓主）。
- 支持轨迹重命名与删除。

#### 3.5.2 轨迹展示与导航
- 管理区展示轨迹列表，包含名称、关联成员、距离、轨迹点数。
- 支持一键导航，跳转网页地图到轨迹终点坐标。
- 轨迹数据持久化保存，刷新页面后可继续使用。

### 3.6 族谱照片 OCR 导入（Stage2 P0）
#### 3.6.1 图片上传与预处理
- 支持上传族谱照片（jpg/png/heic），移动端支持调用相机拍照。
- 前端自动压缩图片（宽度上限 2000px）后再进行识别。

#### 3.6.2 OCR 识别与解析
- 调用 OCR 服务识别文字内容，提取姓名、父亲、母亲、配偶等字段。
- 采用规则解析将文本映射为可编辑的临时成员数据。

#### 3.6.3 预览编辑与去重导入
- 提供表格式预览工作台，支持增删改识别结果。
- 导入前提示重复候选，用户选择策略后批量导入族谱。

### 3.7 本地存储升级（SQLite 第一期）
#### 3.7.1 存储目标
- 存储主路径由 localStorage 升级为 SQLite（SQL.js），并使用 IndexedDB 持久化数据库二进制。
- 保持离线可用，不依赖后端服务。
- 保留 JSON 导入导出能力，同时新增 SQLite 文件导入导出能力。

#### 3.7.2 兼容与迁移
- 兼容历史 localStorage 数据：首次启用 SQLite 时自动迁移或提示迁移。
- 支持灰度切换与降级开关：异常时可回退 localStorage。
- 升级过程中支持双写（SQLite + localStorage）以降低发布风险。

#### 3.7.3 SQLite 核心模型（一期）
- members：成员主表（id/name/parent_id/gender/spouse_ids/created_at/updated_at）。
- tracks：轨迹表（points、起终点、统计信息 JSON 化存储）。
- metadata：元数据表（schema_version/next_id/next_track_id/last_sync_time 等）。

## 7. 交互流程与约束

### 7.1 新增成员流程
1. 用户在右侧管理面板输入姓名、选择父亲（可选）和性别。
2. 点击“保存”按钮提交。
3. 系统校验姓名非空，校验通过后创建新成员并分配自增 ID。
4. 保存至 SQLite（可选双写 localStorage），刷新树图、成员列表和父级选项。
5. 新增成功后自动选中新成员，便于继续查看或编辑。

### 7.2 编辑成员流程
1. 用户通过列表或详情进入编辑态，复用右侧管理面板（非弹窗）。
2. 修改姓名、父亲或性别后点击“保存”。
3. 系统进行姓名非空、自引用和后代检测校验。
4. 校验通过则更新成员对象并刷新界面。
5. 若校验失败则提示错误原因并保持当前数据不变。

### 7.3 删除成员流程
1. 用户点击删除按钮。
2. 弹出确认框：“确认删除 [姓名] 及其全部后代吗？”。
3. 用户确认后，递归获取所有后代 ID，从成员数组中过滤删除（级联删除）。
4. 保存并刷新界面，提示“已删除 X 位成员”。

### 7.4 循环依赖检测规则
当编辑成员的父亲时，系统会检查：若将 A 的父亲设为 B，则 B 不能是 A 自身，也不能是 A 的后代。检测算法为基于子树遍历（DFS/栈）判断候选父节点是否位于 A 的后代集合中，若命中则拒绝操作。

### 7.5 JSON 导入导出流程
1. 用户点击“导入 JSON”后选择本地 JSON 文件。
2. 系统先弹出覆盖确认：“导入将覆盖当前数据，是否继续？”。
3. 用户确认后执行导入；成功提示“导入成功”，失败提示具体错误。
4. 用户可点击“导出 JSON”下载当前数据备份。
5. 导入结束后会清空文件输入状态，避免重复触发。

### 7.5.1 SQLite 导入导出流程
1. 用户点击“导出 SQLite”下载当前数据库文件（.sqlite/.db）。
2. 用户可选择 SQLite 文件导入，导入前执行覆盖确认。
3. 导入后系统替换当前数据库并刷新树图、列表、详情。
4. 导入失败时提示错误并保持现有数据不变。

### 7.5.2 首次迁移流程（localStorage -> SQLite）
1. 页面启动时优先初始化 SQLite 与 IndexedDB。
2. 若 SQLite 为空且检测到 localStorage 数据，则执行一次性迁移。
3. 迁移成功后写入迁移标记，并进入 SQLite 主路径。
4. 迁移失败时回退至 localStorage 并提示用户导出备份。

### 7.6 OCR 导入流程
1. 用户点击“从族谱照片导入”，上传图片或使用拍照入口。
2. 系统完成图片压缩并执行 OCR 识别。
3. 前端解析识别文本并展示可编辑预览表格。
4. 用户可修正字段并处理重复候选（跳过/新增）。
5. 点击导入后批量写入成员与关系并刷新树图。

## 8. 技术实现说明

| 技术栈 | 用途 |
|-------|------|
| Vue 3 | 组件化界面与响应式状态驱动 |
| TypeScript | 类型约束与业务逻辑可靠性 |
| Vite | 本地开发与构建工具链 |
| ECharts ^5.6.0 | 族谱树可视化渲染与交互 |
| SQL.js + SQLite | 结构化本地数据存储 |
| IndexedDB | SQLite 数据库二进制持久化 |
| localStorage | 兼容迁移与回滚兜底 |

### 关键技术点
- **树图数据转换**：将扁平成员数组转换为嵌套的树结构，通过虚拟根节点统一挂载。
- **自适应**：使用 `@media` 查询配合 Flex 布局。
- **移动端触摸优化**：ECharts 默认支持触摸事件，通过 `roam: true` 启用缩放拖拽。
- **关系校验**：更新父节点时执行自引用与后代检测，防止关系成环。
- **导入导出闭环**：支持 JSON/SQLite 文件导入、覆盖确认与导出备份。
- **存储升级兼容**：支持 localStorage 到 SQLite 一次性迁移、双写过渡和降级回退。

## 9. 异常场景处理

| 场景 | 处理方式 |
|------|---------|
| 添加成员时姓名为空 | 弹出 alert 提示“姓名不能为空”，不执行添加 |
| 编辑时姓名为空 | 同样 alert 提示，阻止保存 |
| 将父亲设为自己 | alert 提示“父节点不能是本人或后代” |
| 形成循环依赖 | alert 提示“父节点不能是本人或后代” |
| 删除成员不存在 | 返回“成员不存在”，不执行删除 |
| 导入 JSON 格式无效 | 提示“导入失败：JSON 格式无效” |
| 导入数据结构错误 | 提示“导入失败：数据结构错误” |
| 导入后无有效成员数据 | 提示“导入失败：未找到有效成员数据” |
| 导入数据成员 ID 重复 | 提示“导入失败：成员 ID 重复” |
| 导入覆盖确认未通过 | 取消导入，保持原数据不变 |
| 父亲 ID 在编辑时选择无效（如已被删除） | 程序自动转为 null，即设为无父亲 |
| GPX 文件格式无效 | 提示“GPX 解析失败：文件格式无效” |
| GPX 文件过大（超过 10MB） | 提示“GPX 文件过大（上限 10MB）” |
| GPX 轨迹点不足 | 提示“GPX 解析失败：轨迹点不足（至少 2 个）” |
| OCR 识别服务失败 | 提示“OCR 识别失败”，用户可重试 |
| OCR 结果无有效姓名 | 提示“OCR 导入失败：无有效成员姓名” |
| OCR 导入发现重复候选 | 提示重复处理策略，由用户选择跳过或新增 |
| SQLite 初始化失败 | 提示“本地数据库初始化失败”，支持重试 |
| SQLite 导入失败 | 提示“SQLite 导入失败”，保持原数据不变 |
| 首次迁移失败 | 回退 localStorage 并提示导出备份 |

## 10. Stage2 迭代路线（摘要）

### M1（P0）：关系模型升级与数据安全
- 增加配偶关系与双向一致性维护。
- 删除成员时清理配偶引用。
- 导入导出加入版本校验、迁移与预检查。

### M1.5（P0，与 M1 并行）：扫墓轨迹最小闭环
- 对齐 `docs/navigation.MD`：TR-01~TR-04、MV-01~MV-02、NA-01。
- 交付最小闭环：GPX 上传解析、轨迹管理、导航跳转。

### M1.8（P0，与 M1.5 并行）：OCR 识别导入最小闭环
- 对齐 `docs/OCR.MD`：UP-01/UP-02/UP-04、OC-01/OC-02、PA-01/PA-02/PA-04、IM-01/IM-02。

## 附录：术语表

- **族谱**：记载家族世系和重要人物事迹的图册。
- **始祖**：家族中无父亲记录的成员，代表一个分支的起点。
- **虚拟根节点**：在树图中为了统一展示所有顶级成员而创建的非实际节点，不存储于数据中。
- **循环依赖**：将后代设为祖先的父亲，导致关系环路，属于逻辑错误。
