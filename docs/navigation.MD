# 扫墓轨迹功能需求文档（PRD）

## 1. 功能概述
在现有电子族谱系统基础上，新增**扫墓轨迹管理**模块。用户可上传 GPX 格式的轨迹文件，系统在地图上展示该轨迹，并提供导航功能，便于家族成员在实地扫墓时参考路线。

---

## 2. 用户故事
- **作为** 家族管理员  
- **我想要** 上传往年扫墓的 GPX 轨迹文件  
- **以便** 让其他成员了解墓地位置和最佳路线

- **作为** 普通家族成员  
- **我想要** 在成员详情页或独立地图页查看扫墓轨迹  
- **以便** 在清明节等时节顺利找到墓地并导航前往

---

## 3. 功能需求

### 3.1 轨迹上传与管理
| ID | 功能 | 描述 |
|----|------|------|
| TR-01 | 上传 GPX 文件 | 支持 .gpx 格式文件，解析后提取经纬度点集、元数据（如名称、时间） |
| TR-02 | 关联成员 | 可将轨迹与特定家族成员（墓主）绑定，或作为独立扫墓记录 |
| TR-03 | 编辑/删除 | 支持修改轨迹名称、关联成员，删除无用轨迹 |
| TR-04 | 轨迹列表 | 展示所有已上传轨迹，按时间或关联成员排序 |

### 3.2 轨迹可视化
| ID | 功能 | 描述 |
|----|------|------|
| MV-01 | 地图展示 | 使用地图组件（如 Leaflet + OpenStreetMap）展示轨迹线，可缩放、拖拽 |
| MV-02 | 起终点标注 | 在地图上用图标标注轨迹起点和终点（墓地位置） |
| MV-03 | 高程/距离信息 | 显示轨迹总距离、预计步行/驾车时间、高程变化（可选） |

### 3.3 导航功能
| ID | 功能 | 描述 |
|----|------|------|
| NA-01 | 一键导航 | 点击“导航”按钮，调用手机原生地图（Apple Maps / Google Maps / 高德地图）以终点为目的地 |
| NA-02 | 路线规划 | 若当前用户允许获取位置，可显示从当前位置到墓地的最佳路线 |
| NA-03 | 分享轨迹 | 生成轨迹链接或二维码，方便其他成员在手机上打开并导航 |

---

## 4. 非功能需求

| 类别 | 要求 |
|------|------|
| 兼容性 | 支持 Chrome、Safari、Edge 等现代浏览器，移动端触摸友好 |
| 性能 | GPX 文件大小不超过 10MB，解析时间 < 2 秒，地图加载流畅 |
| 安全性 | 仅本地存储，不上传任何数据；GPX 文件解析在前端完成，不依赖后端 |
| 离线可用 | 地图瓦片需考虑离线缓存（可选），核心功能不依赖互联网（除地图瓦片外） |

---

## 5. 数据模型

### 5.1 轨迹对象（Track）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | number | 唯一标识 |
| name | string | 轨迹名称（默认使用文件名或自定义） |
| memberId | number/null | 关联的成员 ID（墓主） |
| gpxData | object | 解析后的轨迹数据（坐标数组、距离、高程等） |
| createdAt | timestamp | 上传时间 |

### 5.2 存储方式
- 原有 localStorage 存储 members 数据
- 新增 `tracks` 数组，存储轨迹对象
- GPX 原始文件内容不保存，只保存解析后的 JSON 数据（坐标点集）

---

## 6. 交互流程

### 6.1 上传轨迹
1. 用户进入“扫墓轨迹”管理页面（可在管理端新增选项卡或入口）
2. 点击“上传 GPX 文件”按钮
3. 选择本地 .gpx 文件
4. 系统解析文件，提取坐标、名称、总距离等信息
5. 用户可填写轨迹名称、选择关联的成员
6. 点击“保存”，轨迹数据存入 localStorage，刷新轨迹列表

### 6.2 查看与导航
1. 在成员详情页或轨迹列表页，点击某条轨迹
2. 地图组件加载并绘制轨迹线，自动缩放至合适视野
3. 显示轨迹信息（距离、起点/终点）
4. 点击“导航”按钮：
   - 若为移动设备，调用系统地图应用（URL Scheme），传入终点坐标
   - 若为 PC 端，跳转至在线地图网页（如高德地图、百度地图）并显示路线

## 7. 技术实现建议

### 7.1 GPX 解析
- 使用第三方库如 `gpxparser` 或手写 DOMParser 解析 XML
- 提取 `<trkpt>` 中的 lat/lon，可选 `<ele>` 高程
- 计算总距离（Haversine 公式）

### 7.2 地图组件
- **推荐方案**：Leaflet + OpenStreetMap 瓦片（免费、无 API 密钥）
- **备选**：高德地图 JS API（国内访问稳定，需申请 key）
- 需要在地图上添加轨迹线图层、起点/终点标记

### 7.3 导航调用
- 移动端：使用 `window.open` 打开特定 URL Scheme
  - iOS/Android 通用：`maps:?q=lat,lng`（Apple Maps）
  - Google Maps：`https://www.google.com/maps/dir/?api=1&destination=lat,lng`
  - 高德地图：`https://uri.amap.com/navigation?to=lat,lng`
- PC 端：跳转到高德/百度地图网页版，传入终点坐标

### 7.4 性能优化
- 若轨迹点过多，进行抽稀处理（Douglas-Peucker 算法）
- 使用 Web Worker 解析大文件，避免阻塞 UI

## 8. 界面原型（文字描述）
- **轨迹管理页**：列表展示轨迹卡片，每个卡片包含名称、关联成员、上传时间，右侧有“查看地图”按钮
- **地图详情页**：全屏或半屏地图，顶部显示轨迹信息栏，底部有“导航”按钮和“返回”按钮
- **成员详情页**：在成员信息下方增加“扫墓轨迹”区块，若有关联轨迹则显示链接，否则显示“暂无”
