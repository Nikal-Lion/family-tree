# 族谱照片 OCR 识别导入功能需求文档（PRD）

## 1. 功能概述
为方便用户将纸质族谱快速数字化，本功能支持用户上传族谱照片（或扫描件），通过 OCR（光学字符识别）技术自动提取照片中的文字信息，解析出成员姓名、父亲、母亲、生卒年、配偶等关键字段，并转换为系统可识别的结构化数据，用户确认后一键导入到电子族谱中。

---

## 2. 用户故事
- **作为** 家族管理员  
- **我希望** 拍摄或上传纸质族谱的照片  
- **以便** 系统能自动识别并导入成员信息，省去逐条手动录入的繁琐工作。

- **作为** 普通用户  
- **我希望** 在导入后能人工核对并修正识别结果  
- **以便** 确保族谱数据的准确性。

---

## 3. 功能需求

### 3.1 图片上传与预处理
| ID | 功能 | 描述 |
|----|------|------|
| UP-01 | 选择图片 | 支持从本地选择图片（jpg/png/heic），支持多选或单张上传 |
| UP-02 | 相机拍照 | 移动端支持调用摄像头直接拍摄族谱页面 |
| UP-03 | 图片裁剪/旋转 | 提供简单的图片调整工具，用户可裁剪边缘、旋转校正倾斜 |
| UP-04 | 图片压缩 | 前端压缩图片至合适尺寸（如宽度不超过 2000px），减少 OCR 请求负载 |

### 3.2 OCR 文字识别
| ID | 功能 | 描述 |
|----|------|------|
| OC-01 | 调用 OCR 服务 | 支持对接第三方 OCR API（如百度 OCR、腾讯 OCR、阿里 OCR）或本地 Tesseract.js |
| OC-02 | 文字区域识别 | 识别图片中所有文字块，输出文字内容及位置（可选） |
| OC-03 | 表格结构识别 | 若族谱采用表格形式（如世系表），需识别行列关系，将单元格内容对应起来 |

### 3.3 信息解析与映射
| ID | 功能 | 描述 |
|----|------|------|
| PA-01 | 字段提取 | 从识别结果中提取姓名、父亲、母亲、配偶、生卒年、子嗣等字段，支持常见族谱表述（如“娶某氏”、“生子X人”） |
| PA-02 | 关系推理 | 根据父子、夫妻关系自动生成成员间的关联（parentId 和配偶关系） |
| PA-03 | 人工映射 | 若识别不确定，提供界面让用户手动匹配字段（如拖拽文字到对应属性） |
| PA-04 | 预览与编辑 | 将解析出的成员列表以表格形式展示，支持增删改、调整关系，确认无误后导入 |

### 3.4 导入与去重
| ID | 功能 | 描述 |
|----|------|------|
| IM-01 | 去重检测 | 检测待导入成员是否与现有成员重复（姓名+父亲或相似度），提示用户合并或覆盖 |
| IM-02 | 批量导入 | 一键将确认后的所有成员及关系写入系统，刷新树图和列表 |
| IM-03 | 导入历史 | 记录每次导入的原始图片和解析结果，便于回溯 |

---

## 4. 非功能需求

| 类别 | 要求 |
|------|------|
| 识别准确率 | 针对清晰、规范印刷体的族谱，姓名识别准确率 ≥ 90%；手写体准确率 ≥ 70% |
| 响应时间 | 单张图片 OCR 处理时间 ≤ 5 秒（取决于网络和 API） |
| 隐私保护 | 若使用第三方 API，需告知用户数据将上传至云端；也可提供本地 Tesseract.js 模式（离线，但速度较慢） |
| 多语言 | 支持繁体中文、简体中文识别 |
| 兼容性 | 所有现代浏览器，移动端拍照功能需 HTTPS 环境 |

## 5. 数据模型扩展

为支持 OCR 导入功能，需增加临时数据结构用于存储识别结果和用户修正。

### 5.1 临时成员对象（TempMember）
| 字段 | 类型 | 说明 |
|------|------|------|
| tempId | string | 临时唯一标识 |
| name | string | 姓名（识别结果） |
| fatherName | string | 父亲姓名（文本） |
| motherName | string | 母亲姓名 |
| spouseName | string | 配偶姓名 |
| birthYear | string | 生年（如“光绪二十年”） |
| deathYear | string | 卒年 |
| gender | string | 性别（可推断） |
| rawText | string | 原始识别文本，供调试 |

### 5.2 解析结果存储
- 导入过程产生的临时数据可暂存在 sessionStorage 或内存中，导入成功后清除。

## 6. 交互流程

### 6.1 启动导入
1. 用户在管理端点击“从族谱照片导入”按钮。
2. 弹出导入向导，提示选择图片或拍照。

### 6.2 图片处理
1. 用户选择图片后，显示图片预览，提供裁剪/旋转工具。
2. 用户确认后，系统对图片进行压缩，并发送至 OCR 服务。

### 6.3 识别与解析
1. 系统显示“正在识别…”加载状态。
2. 获取 OCR 结果后，前端解析模块提取文本并尝试识别结构化数据。
3. 若解析成功，展示成员列表预览界面；若失败，提示用户手动输入或调整图片。

### 6.4 用户确认
1. 预览界面以表格形式列出识别出的所有成员，每行包含姓名、父亲、母亲、生卒等。
2. 用户可编辑任意单元格，也可删除行、添加行。
3. 对于关系不明确的成员（如父亲姓名在现有成员中不存在），提供选项“新建该父亲”或“忽略”。
4. 点击“导入族谱”按钮。

### 6.5 去重与合并
1. 系统检查每个待导入成员是否与现有成员重复（依据姓名+父亲相似度）。
2. 若重复，弹出合并选项：覆盖现有信息、新建为不同人、跳过。
3. 最终确认后，将新成员写入 members 数组，并建立 parentId 关联。

### 6.6 完成
1. 刷新族谱树和成员列表，提示“导入成功 X 人”。
2. 可选：保存本次导入的原始图片和解析日志。

## 7. 技术实现建议

### 7.1 OCR 服务选型

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| 百度 OCR（通用文字识别） | 准确率高，支持表格识别，中文友好 | 需申请 API Key，有免费额度限制 | 推荐生产环境 |
| 腾讯 OCR | 同上 | 同上 | 备选 |
| Tesseract.js | 完全本地，无需 API Key，隐私安全 | 准确率较低，处理速度慢，对图片质量要求高 | 离线或演示环境 |
| 阿里 OCR | 准确率高 | 需付费 | 企业级应用 |

建议优先使用百度 OCR 通用文字识别（高精度版），并配置前端加密传输。

### 7.2 前端 OCR 调用封装
- 使用 `axios` 或 `fetch` 向代理服务（或直接调用 API）发送图片 base64。
- 为防止暴露 API Key，建议通过 Cloudflare Worker 或简单后端转发（但本系统为纯前端，可接受 API Key 暴露风险较小，但需注意额度控制）。

### 7.3 解析规则
- 姓名正则：匹配常见中文姓名（2-4字）。
- 父亲/母亲：提取“父：xxx”、“母：xxx”模式。
- 生卒年：提取“生于…”、“卒于…”或数字年份。
- 配偶：提取“配某氏”、“娶某氏”。
- 子嗣：提取“子N人”或具体子嗣姓名列表。

可使用 `正则表达式` + 简单 NLP 库（如 `compromise`）增强解析。

### 7.4 去重算法
- 比较待导入成员与现有成员的姓名和父亲姓名相似度（可使用 Levenshtein 距离或拼音匹配）。
- 若相似度 > 0.8，则视为可能重复，提示用户。

## 8. 界面原型（文字描述）

- **导入入口**：管理端新增“📸 从照片导入”按钮（与“新增族人”并列）。
- **图片上传区**：拖拽或点击上传图片，支持多张；移动端显示相机调用。
- **图片编辑区**：简单裁剪工具，支持旋转、缩放。
- **识别中**：进度条 + 提示语（“系统正在识别族谱文字，请稍候…”）。
- **结果预览表**：类似 Excel 表格，各字段可编辑，顶部显示“添加行”按钮。
- **去重提示框**：弹窗列出重复成员，供选择处理方式。
- **成功提示**：Toast 或弹窗显示导入结果。

## 9. 验收标准

- [ ] 支持上传 jpg/png 图片，移动端可拍照。
- [ ] 能够调用 OCR API 并返回文字结果。
- [ ] 能从识别文本中正确提取至少 80% 的姓名、父亲关系。
- [ ] 预览表格支持增删改行，可调整字段。
- [ ] 导入前能检测重复成员并提供合并选项。
- [ ] 导入后族谱树和成员列表正确更新。
- [ ] 整个流程在 4G 网络下单张图片 < 10 秒完成。
- [ ] 错误处理：OCR 失败时提示用户重试或手动输入。

---

## 10. 后续扩展

- 支持批量导入多张图片（多页族谱）并自动合并结果。
- 增加手写体族谱的专用识别模型（需额外训练）。
- 导入后可生成导入报告，列出未识别项供用户补充。
- 集成 LLM（大语言模型）进行更智能的语义解析，理解族谱中的古文表述。
